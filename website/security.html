  • Session level data is encrypted with a long random string generated each time you login.
  • Persistent data stored between logins is encrypted with your clear text password, which is obviously not stored anywhere on the server or in the session.
  • All cookies are session level, HTTP only, and have the secure flag, path, and domain values set.
  • All encryption is AES compliant using PHP MCRYPT with tunable knobs.
  • TLS everywhere - forced TLS for browsers, and TLS/STARTTLS support for IMAP, POP3, and SMTP connections.
  • Oauth2 login for Gmail and Outlook servers supported.
  • HTTP request header fingerprinting, site/user specific page keys, and dynamic session ids.
  • Security related HTTP headers like X-XSS-Protection, X-Content-Type-Options, and Content-Security-Policy.
  • Two factor authentication module set that uses a free SMS based 2FA service from Swipeidentity.
  • Only white-listed and typed/sanitized input is allowed. Modules must pre-define input in order to have access to it.
  • Modules have no access to PHP super-globals, and there are no globally scoped variables at all (there are a few static class instances with private data structures that use getter/setter methods).
  • PHP ini settings are tightened up at runtime for extra security, including open basedir and session best practices.
  • No writable files or directories are used inside the web-server document root, and only 3 files need to be inside the document root to run the program.
  • Optional local DB based authentication uses PBKDF2 hashing.
  • Easy output escaping inside modules for potentially dangerous content.
  • Configurable idle timeout module set that ends a session after a predetermined amount of time.
  • Module design enforces a separation of input processing and output formatting, and encourages concise methods, which makes the code easier to debug and audit.
  • 100% unit test coverage with PHPUnit for the application framework, with a cyclomatic complexity maximum for classes of 38, and a method complexity max of 11 (http://cypht.org/docs/test_coverage/dashboard.html).
