Home Features Security Code Install
  • Session level data is encrypted with a long random string generated each time you login.
  • Persistant data stored between logins is encrypted with your clear text password, which is not stored anywhere on the server or in the session.
  • All cookies are session level, HTTP only, and have the secure flag, path, and domain values set.
  • All encryption is AES compliant using PHP MCRYPT with tunable knobs.
  • TLS everwhere - forced TLS for browsers, and TLS/STARTTLS support for IMAP, POP3, and SMTP connections.
  • HTTP request header fingerprinting and dynamic session ids.
  • Built in nonce checking for all HTTP POST requests, including AJAX based requests. It is not possible for modules to override the nonce check.
  • Security related HTTP headers like X-XSS-Protection, X-Content-Type-Options, and Content-Security-Policy.
  • An optional two factor authentication module that uses a free SMS 2fa service from Swipeidentity.
  • Only white-listed and typed/sanitized input is allowed. Modules must pre-define input in order to have access to it.
  • Modules have no access to PHP super-globals, and there are no globally scoped variables at all (there are a few static class instances with private data structures that use getter/setter methods).
  • PHP ini settings are tightened up for extra security, including open basedir and session best practices.
  • No writable files or directories are used inside the webserver document root, and only 3 files need to be insde the document root to run the program.
  • Optional local DB based authentication uses PBKDF2 hashing.
  • Easy ouput escaping inside modules for potentially dangerous content.
  • Configurable idle timeout module set that ends a session after a predetermined amount of idle time.
  • Module design enforces a separation of input processing and output formatting, and encourages concise methods, which makes the code easier to debug and audit.
